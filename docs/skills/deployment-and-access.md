# 運用・デプロイスキルまとめ

このドキュメントは、任意プロジェクトの本番環境（マネージドコンテナ実行基盤）に対するデプロイ、マイグレーション、およびアクセス権限（公開/非公開）の管理をコマンドで再現するためのスキル集です。

---

## 🚀 デプロイと自動マイグレーション

バックエンドの起動時に自動でデータベースマイグレーションが走る仕組みになっています。そのため、最新のコード（スキーマ変更含む）を反映させるには、**バックエンドをデプロイするだけ**で自動的にマイグレーションまで完了します。

以下のコマンド群（シェルスクリプト）を用意しました。

### 📌 全体のデプロイ（フロント＆バック）
```bash
./scripts/commands/deploy.sh all
```

### 📌 バックエンドのみのデプロイ（※マイグレーションも同時実行）
データベースの構造変更や、APIロジックを変更した場合はこちら。
```bash
./scripts/commands/deploy.sh backend
```

### 📌 フロントエンドのみのデプロイ
UIの変更など、React側のコードを変更した場合はこちら。
```bash
./scripts/commands/deploy.sh frontend
```

> [!NOTE]
> デプロイスクリプト内部には、マネージドDB接続に必要なオプション（例: DB接続設定フラグ）が標準装備されている想定です。

---

## 🌍 サイトの一般公開 / 🛡️ 非公開の切り替え

アクセス制御の設定により、実行基盤上で未認証ユーザーのアクセス可否を切り替えられます。フロント側からバックエンドAPIをコールする構成では、設定変更時は**両方のサービス**を同時に切り替えます。切り替えは以下の専用スクリプトで完了します。

### 🔓 サイトを「一般公開」する（全ての人にアクセスを許可）
```bash
./scripts/commands/toggle-public.sh public
```

### 🔒 サイトを「非公開」にする（Google認証済みの管理者等のみに制限）
```bash
./scripts/commands/toggle-public.sh private
```

---

これらのスクリプトは `scripts/` フォルダに配置し、実行権限を付与して使います。使用するクラウドCLIの認証が通っているターミナル環境であれば、このコマンド群でインフラ操作を実行できます。
